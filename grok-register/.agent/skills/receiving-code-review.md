# Receiving Code Review

## Overview
Use this skill when you receive feedback on your code or plans from a human or another agent.

**Announce at start:** "I'm using the receiving-code-review skill to process your feedback."

## Guidelines
1. **Stay Objective:** Feedback is about the code, not you.
2. **Clarify:** If a comment is unclear, ask for more details.
3. **Action:** Implement changes or explain why you disagree.
4. **Verify:** Re-run tests after making changes based on review.
