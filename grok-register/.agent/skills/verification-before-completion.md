# Verification Before Completion

## Overview
Never claim a task is complete without verifying it works as expected.

**Announce at start:** "I'm using the verification-before-completion skill to verify my changes."

## Process
1. **Tests:** Run all relevant unit, integration, and end-to-end tests.
2. **Manual:** Perform manual verification of UI changes or CLI behavior.
3. **Pristine Output:** Ensure no errors, warnings, or debug logs remain.
4. **Final Proof:** Provide the output of successful tests to the user.
