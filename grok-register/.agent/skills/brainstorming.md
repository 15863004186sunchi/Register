# Brainstorming

## Overview
Brainstorming is the first step of any complex task. Use this skill to explore the problem space, identify requirements, and design the high-level architecture before writing any implementation plans.

**Announce at start:** "I'm using the brainstorming skill to explore the problem space and design the solution."

## Goals
1. **Understand Requirements:** Clarify user intent and identify missing info.
2. **Explore Options:** Consider multiple approaches, not just the first one.
3. **Design Architecture:** Choose patterns, technologies, and component boundaries.
4. **Define Success:** How will we know it works?

## Process
1. **Research:** Read relevant code, docs, and existing implementations.
2. **Ideate:** List potential solutions with pros/cons.
3. **Decide:** Choose the best approach and justify it.
4. **Draft Spec:** Create a high-level specification document.

## Output
Save the brainstorming results to `docs/superpowers/brainstorming/YYYY-MM-DD-<topic>.md`.
